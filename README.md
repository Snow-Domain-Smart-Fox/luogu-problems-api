# Luogu Problems API

可直接部署在 Vercel 平台的编程题目 API 服务。

## 功能特性

1. **RESTful API**: 提供 `/api/all` 接口返回所有题目数据
2. **定时更新**: 定期自动执行数据库更新（Vercel Cron Jobs）
3. **Supabase 数据库**: 使用 Supabase Postgres 进行数据存储
4. **智能爬取**: 按页码顺序爬取，每页后等待 0.5 秒，避免请求过快
5. **断点续爬**: 自动记录爬取进度，超时后自动恢复，避免重复爬取

## 超时自动恢复机制

### 问题背景
Vercel Serverless Functions 有严格的超时限制：
- **Hobby 计划**: 最大 10 秒
- **Pro 计划**: 最大 60 秒
- **最大硬限制**: 300 秒（5 分钟）

完整爬取所有题目需要数小时，因此项目实现了**分页爬取 + 断点续爬**机制。

### 工作原理

1. **分页爬取**: 每次请求只爬取**一页**数据（约 50 道题），耗时约 1-2 秒
2. **状态记录**: 使用 Supabase `crawl_tasks` 表记录当前进度：
   - 当前题型（luogu/CF/SP/UVA/AT）
   - 当前页码
   - 总页数
   - 已爬取题目数
   - 任务状态（pending/running/completed/failed）
3. **自动恢复**: Vercel Cron 每 5 分钟自动触发一次 `/api/crawl-page` 端点
   - 如果上次请求超时，Cron 会自动检测到未完成的任务
   - 从断点位置继续爬取下一页
   - 完成一种题型后自动切换到下一种题型
4. **完成检测**: 当所有题型的所有页面都爬取完成后，标记任务为 completed

### Cron 配置
- **`/api/cron`**: 每月 1 日执行，用于启动新的爬取任务或清空数据
- **`/api/crawl-page`**: 每 5 分钟执行一次，用于分页爬取和超时恢复

### 优势
- ✅ 不受 Vercel 超时限制影响
- ✅ 自动从断点恢复，无需手动干预
- ✅ 避免重复爬取，提高效率
- ✅ 实时监控进度，可查询当前状态

## 智能缓存机制

为了节省 API 请求并提高效率，系统实现了智能缓存：

### 缓存内容

1. **总页数缓存**
   - 每种题型的总页数会被缓存到 `type_page_totals` 字段
   - 首次获取后，后续直接使用缓存值
   - 减少重复请求洛谷 API

2. **已爬取页面记录**
   - 使用 `crawled_pages` 数组记录已爬取的页面
   - 格式：`["luogu:1", "luogu:2", "CF:1", ...]`
   - 下次触发时自动跳过已爬取的页面

### 强制刷新

如果需要重新爬取（例如数据更新），可以使用 `force=true` 参数：

```bash
# 强制刷新，清空所有缓存和记录
GET /api/crawl-page?force=true

# 或清空数据库后重新爬取
GET /api/cron?clear=true&force=true
```

**注意**：强制刷新会：
- 删除旧的 `crawl_tasks` 记录
- 清空 `crawled_pages` 缓存
- 重新爬取所有页面

### 跳过逻辑

当访问 `/api/crawl-page` 时：
1. 检查当前页面是否已在 `crawled_pages` 中
2. 如果已爬取，直接跳过并移动到下一页
3. 返回响应中包含 `skipped: true` 标识

示例响应：
```json
{
  "message": "Skipped luogu page 5 (already crawled)",
  "skipped": true,
  "nextPage": 6,
  "nextType": "luogu"
}
```

## 智能冷却机制（30 天定期更新）

为避免任务完成后继续浪费 Cron 资源，系统实现了智能冷却机制：

### 工作原理

1. **任务完成后**：系统进入 30 天冷却期
2. **冷却期内**：每 7 分钟的 Cron 触发会快速返回，不创建新任务
3. **冷却期后**：自动创建新任务，开始下一轮爬取（定期更新数据）

### 时间线示例

```
Day 0  - 爬取完成 → 进入冷却期
Day 1  - Cron 触发 → 返回 "Completed 1 days ago"
Day 7  - Cron 触发 → 返回 "Completed 7 days ago"
...
Day 30 - Cron 触发 → 冷却期结束，自动开始新任务
Day 31 - 开始爬取新数据（如果有更新）
```

### 响应示例

**冷却期内**：
```json
{
  "status": "completed",
  "message": "All problems crawled successfully. Completed 15 days ago.",
  "completedAt": "2024-01-01T00:00:00.000Z",
  "daysSinceCompletion": 15,
  "nextCheck": "30 days after completion",
  "note": "Use ?force=true to start a new crawl immediately"
}
```

**查询进度时显示**：
```json
{
  "status": "completed",
  "message": "Crawl completed successfully! 15 days ago.",
  "cooldown": {
    "daysSinceCompletion": 15,
    "daysRemaining": 15,
    "nextAutoUpdate": "15 days"
  }
}
```

### 手动触发

如果不想等待 30 天，可以手动触发：

```bash
# 立即开始新任务（忽略冷却期）
GET /api/crawl-page?force=true
```

### 优势

- ✅ **节省资源**：冷却期内不执行爬取
- ✅ **自动更新**：30 天后自动开始新任务
- ✅ **灵活控制**：支持手动强制刷新
- ✅ **状态透明**：随时查询冷却进度

## 完整部署指南

### 前置要求

- 一个 GitHub 账户
- 一个 Vercel 账户（可免费注册）
- 基本的 Git 操作知识

### 详细部署步骤

#### 第一步：准备项目代码

1. **克隆或下载此项目**
   ```bash
   git clone <your-repository-url>
   ```

2. **初始化 Git 仓库（如果还没有）**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   ```

3. **推送到 GitHub**
   - 在 GitHub 上创建一个新的仓库
   - 按照 GitHub 提示将本地代码推送到远程仓库

#### 第二步：在 Vercel 中导入项目

1. **登录 Vercel**
   - 访问 [vercel.com](https://vercel.com)
   - 使用 GitHub 账户登录

2. **导入项目**
   - 点击 **Add New Project**
   - 选择刚才创建的 GitHub 仓库
   - 点击 **Import**

3. **配置项目（可选）**
   - **Project Name**: 输入项目名称（例如：luogu-problems-api）
   - **Framework Preset**: Vercel 会自动检测为 Other
   - **Root Directory**: 保持默认（如果项目在根目录）
   - **Build Command**: 留空
   - **Output Directory**: 留空

#### 第三步：配置 Supabase 数据库

1. **创建 Supabase 项目**
   - 访问 [supabase.com](https://supabase.com) 并注册/登录
   - 点击 **New Project**
   - 填写项目信息：
     - **Name**: luogu-problems-api
     - **Database Password**: 设置一个强密码（请妥善保存）
     - **Region**: 选择离你最近的区域
   - 点击 **Create new project**
   - 等待项目创建完成（通常需要 1-2 分钟）

2. **创建数据库表**
   
   **表 1: problems（题目数据）**
   - 在 Supabase Dashboard 中，点击左侧菜单的 **Table Editor**
   - 点击 **New table**
   - 填写表信息：
     - **Name**: problems
     - **Enable Row Level Security (RLS)**: 关闭
   - 添加以下列：
     | 列名 | 类型 | 是否可空 | 默认值 | 说明 |
     |------|------|----------|--------|------|
     | id | text | ❌ 否 | - | 主键（题目 ID） |
     | difficulty | int8 | ✅ 是 | null | 难度 |
     | updated_at | timestamp | ❌ 否 | now() | 更新时间 |
   - 点击 **Save** 创建表
   
   **表 2: crawl_tasks（爬取任务管理）**
   - 再次点击 **New table**
   - 填写表信息：
     - **Name**: crawl_tasks
     - **Enable Row Level Security (RLS)**: 关闭
   - 添加以下列：
     | 列名 | 类型 | 是否可空 | 默认值 | 说明 |
     |------|------|----------|--------|------|
     | id | int8 | ❌ 否 | auto increment | 主键 |
     | status | text | ❌ 否 | 'pending' | 状态：pending/running/completed/failed |
     | current_type | text | ✅ 是 | null | 当前题型：luogu/CF/SP/UVA/AT |
     | current_page | int8 | ❌ 否 | 0 | 当前页码 |
     | total_pages | int8 | ❌ 否 | 0 | 总页数 |
     | problems_crawled | int8 | ❌ 否 | 0 | 已爬取题目数 |
     | error_message | text | ✅ 是 | null | 错误信息 |
     | created_at | timestamp | ❌ 否 | now() | 创建时间 |
     | updated_at | timestamp | ❌ 否 | now() | 更新时间 |
   - 点击 **Save** 创建表

3. **获取 Supabase 连接信息**
   - 在 Supabase Dashboard 中，点击左侧菜单的 **Project Settings**
   - 点击 **API**
   - 复制以下信息：
     - **Project URL** (Project URL)
     - **service_role** (Service role secret)

4. **配置 Vercel 环境变量**
   - 回到 Vercel 项目导入页面
   - 点击 **Environment Variables** 标签
   - 添加以下两个环境变量：
     1. **Name**: `SUPABASE_URL`
        **Value**: 粘贴刚才复制的 Project URL
        **Environment**: 勾选 Production, Preview, Development
        点击 **Save**
     2. **Name**: `SUPABASE_SERVICE_ROLE_KEY`
        **Value**: 粘贴刚才复制的 service_role
        **Environment**: 勾选 Production, Preview, Development
        点击 **Save**

#### 第四步：配置 Cron 密钥（可选但推荐）

1. **添加 Cron 密钥（推荐）**
   - 在项目导入页面，点击 **Environment Variables** 标签
   - 点击 **Add New**
   - **Name**: `CRON_SECRET`
   - **Value**: 输入一个强密码（例如：`my-strong-cron-secret-2024!`）
   - **Environment**: 勾选 **Production**, **Preview**, **Development**
   - 点击 **Save**

#### 第五步：部署项目

1. **开始部署**
   - 在项目导入页面，点击 **Deploy**
   - 等待部署完成（通常需要 1-3 分钟）

2. **验证部署**
   - 部署成功后，点击 **Continue to Dashboard**
   - 点击 **Visit** 查看项目首页
   - 访问 `<your-domain>.vercel.app/api/all` 验证 API 是否正常工作

#### 第六步：验证 Cron Jobs 配置

1. **查看 Cron Jobs**
   - 在 Vercel 项目 Dashboard 中，点击 **Settings**
   - 点击 **Cron Jobs**
   - 你应该能看到一个已配置的 Cron Job：
     - **Path**: `/api/cron`
     - **Schedule**: `0 0 1 * *`（每月 1 日 UTC 0:00）

2. **测试 Cron Job（可选）**
   - 注意：由于安全机制，你无法直接访问 `/api/cron` 端点
   - Cron Job 只会由 Vercel 自动触发

### 常见问题排查

#### 问题 1：部署失败

**可能原因**：
- Supabase 环境变量未正确配置
- 依赖安装失败

**解决方法**：
1. 检查 Vercel 项目设置中的环境变量是否包含 `SUPABASE_URL` 和 `SUPABASE_SERVICE_ROLE_KEY`
2. 确认环境变量值是否正确
3. 查看 Build Logs 了解具体错误
4. 尝试重新部署

#### 问题 2：API 返回空数组

**可能原因**：
- 数据还在爬取中
- 爬取过程出错
- Supabase 表未正确创建

**解决方法**：
1. 等待 10-30 分钟后重试
2. 查看 Runtime Logs 检查是否有错误
3. 确认 Supabase 中是否已创建 `problems` 表
4. 检查表结构是否正确
5. 检查 RLS（Row Level Security）是否已关闭

#### 问题 3：Cron Job 不工作

**可能原因**：
- Cron Job 配置错误
- 安全验证失败

**解决方法**：
1. 检查 Settings > Cron Jobs 中的配置
2. 确认 `vercel.json` 文件中的配置正确
3. 查看 Runtime Logs 检查 Cron Job 执行记录

## API 使用

### 获取所有题目

```
GET /api/all
```

响应示例：

```json
[
  {
    "id": "P1000",
    "difficulty": 1,
    "updated_at": "2024-01-01T00:00:00.000Z"
  }
]
```

### 查询爬取进度

```
GET /api/crawl-status
```

响应示例：

```json
{
  "status": "running",
  "taskId": 1,
  "currentType": "luogu",
  "currentPage": 15,
  "totalPages": 50,
  "problemsCrawled": 700,
  "totalProblemsInDb": 700,
  "progress": "30%",
  "message": "Currently crawling luogu page 15/50"
}
```

**状态说明：**
- `pending`: 等待开始
- `running`: 正在爬取
- `completed`: 已完成
- `failed`: 失败（查看 `errorMessage` 字段）
- `idle`: 无任务

### 手动触发爬取（可选）

```
GET /api/cron?clear=true
```

**参数说明：**
- `clear=true`（可选）: 清空现有数据后重新开始

**注意：** 正常情况下无需手动触发，Cron 会自动执行。

### 健康检查

```
GET /api/health
```

响应：

```json
{
  "status": "ok"
}
```

## 定时任务

项目配置了 Vercel Cron Jobs，在每月 1 日 UTC 0:00（即 UTC+8 8:00）自动执行数据库更新，清空并重新爬取所有题目。

### Cron 安全配置

为了防止手动调用 `/api/cron` 端点，项目使用了 Vercel Cron 的安全验证机制：

1. **请求头验证**: 只有包含 `x-vercel-cron: 1` 请求头的请求才会被接受（Vercel Cron 会自动添加此请求头）
2. **可选的密钥验证**: 可以通过设置 `CRON_SECRET` 环境变量启用额外的密钥验证

#### 配置 Cron 密钥（推荐）

1. 在 Vercel 项目设置中添加环境变量 `CRON_SECRET`，设置一个强密码
2. 在 Vercel 项目的 Cron Jobs 配置中，设置相同的密钥值
3. 系统会验证 `x-vercel-cron-secret` 请求头是否与环境变量匹配
